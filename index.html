Website Overview

Project Summary

M Space Creative is being engaged to help enhance the user experience, user interface and overall design and layout of the current staging site for MSSU. Other requirements are to push enhancements/changes from the staging environment to the new production URL, at mssu.ca.

The current staging site environment at dev.spor-maritime-srap.ca will be used for review and approval of changes made based on the client requirements.

Client Requirements

The client has provided a comprehensive list of requirements to be implemented before site launch. They are listed below:

  • Check that site structure and plug-ins is stable and appropriate, and provide support troubleshooting and fixing any issues that arise
  • Redesign the homepage, review content types and key landing pages to make tweaks to improve design and better align with existing brand guidelines (as needed), and make the styling more consistent throughout the site
  • Perform an accessibility audit, help draft accessibility statement, and make recommendations for future improvements
  • Upload French language content and troubleshoot use of the WPML plug-in
  • Provide updated documentation and training, as needed
  • Support the launch of the new website
  • Other support as needed

Questions for Client

After reviewing the staging site, and looking over the provided requirements for the site overhaul, I have a few questions below.

  1. I've noticed that the multi-language plugin, WPML has an expired account license. Will you be renewing the license on this plugin before we begin work on the site? It is recommended that this subscription is activated before I begin any work, as it will improve workflow and unblock any hurdles with regards to changes that need to be made to the bilingual portion of the site.

    Another instance of an outdated/expired plugin account is with the plugin used for the Twitter Feed. Is there a plan to renew this subscription, or are you wanting me to source another plugin that doesn't require a paid subscription?

  2. For the home page re-design, do you have an idea of what call to actions are needed? Can you provide a list of what content is most important, for example, what content should be displayed above the fold, and how would you like the user to interact with the site so that they can reach the right personnel on MSSU's end when they have inquiries. A content hierarchy doc would be great as a starting point, so that I may provide a wire frame to properly outline the layout for content modeling in the design/development process.

    As well, are you able to define which design elements are to be kept, and which ones should be improved? I have an idea already of how things can be improved, but I would appreciate initial documentation on what the team thinks is working, and what is not with the staging site design.

  3. Is the French content ready for population, or does it still need to be translated in some cases? I'd like to get an idea of how much content there is left to populate, as content population tends to eat up a lot of time.

  4. For site training, do you prefer a manual or an in-person tutorial? My gut is telling me that a tutorial might make more sense, as the bulk of the site has already been built by the other vendor. I don't plan to change things up too much in terms of how pages are created, content is added, etc.

  5. I recall in our meeting, that there was talk of the search functionality perhaps not working as expected. Are you able to elaborate on specific issues that have cropped up with it?

  6. Do you plan on having SEO improvements done to the site? This is something that would be outside of the scope of this project, and SEO is not something that I do directly, but I do have an excellent SEO partner with whom I've worked on many projects in the past with great results. I can put you in touch if needed.
